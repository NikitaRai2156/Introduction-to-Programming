LESSON 1: STRUCTURES, WEBPAGES AND DOCUMENTS

HTML AND WEB BASICS

How the Web Works

The web is fundamentally a group of computers communicating with each other. Whenever a person opens a web page, the computer sends an HTTP request to a server, which then finds the required HTML document and sends it back to the user's computer. Upon the completion of this action, the web browser that the user is using, interprets the web page and displays it on the screen of the output device that the user is using, whether it be a phone or a computer.
For a better understanding of the above concept, click here

HTML

  • HTML stands for HyperText Markup Language.
  • The majority content of the web is formed of HTML documents, which contain text elements that form what you see, and markup elements, that shape up how the page looks.
  • Consult this video for a better overview.

Whitespace

HTML converts all the white space in a single document into a single space. So, several tags exist to carry out the task of making a new line or paragraph.
  • <br> tag is used to create a new line. It's a void tag and thus doesn't require a closing tag. It is an inline element.
  • <p> tag is used to create a new paragraph. It is a block element and requires a closing tag.
  • <span> is an inline element used to create a new line of text.

Some HTML Tags and Attributes

  • Tag Attribute: <tag attribute="value"> contents <tag>
  • Anchor tag: Used to create a link <a href="http://udacity.com"> My Favorite Website <a>
    Anchor tag creates a link such that the words "My Favorite Website" link to the Udacity homepage.
  • Image tag: <img src="url of the image" alt="alternate text">
  • The img tag is a void tag. Also, it is a good practice to put alternate text along with the image.

Inline Elements Vs Block Elements

  • HTML elements can be either of the two types: inline or block.
  • The block elements form an invisible 'block' around the content that they hold.
  • <span>,<b>,<img> etc. are examples of inline elements and <div>,<p> etc. are examples of block elements.

Tags and Elements

  • Every HTML document comprises of HTML elements.
  • The type of each element is indicated to the browser by using different HTML tags.
  • Watch this video for a better understanding of how an HTML document is constructed.

Why Computers are Stupid

Computers are simply machines, and thus they interpret every instruction literally. This means that a small mistake made by a programmer, when implemented literally by the computer, can cause a great deal of trouble.

CREATING STRUCTURED HTML DOCUMENTS

Tree-like Structure of an HTML Document

  • When the HTML elements are nested, they give a tree-like structure to the page, with one element being the parent element of several child elements.
    Refer to the Wikipedia article on Tree Structure for more details.
  • The tree-like structure of a web page is also referred to as the DOM (Document Object Model).
  • CSS (Cascading Style Sheets) are used to apply different styles to the page.
  • CSS makes the code clean and more effiecient or optimized by allowing the user to reuse the same code for different places where the same style is applied.

Boxifying Design

  • The whole structure of any web page can be broken down into a number of boxes.
  • Different boxes will have different content and different class names to identify them.
  • In simpler words, each <div> element creates a new box.

Repitition and Why to Avoid It

  • Repitition leads to errors. Less the repitition of code, lesser will be the errors.
  • Less repitition of code makes it consistent and easy to change if necessary.

Tools Required

  • Test Editors like Sublime Text or Atom.
  • DevTools, or Developer tools. These are different in each browser and help the developer to make temporary changes to their code without changing it permanently.

STYLING THE HTML STRUCTURE

What Is CSS

  • CSS stands for Cascading Style Sheets.
  •              
               .quote {
                   font-size: 55px;
                   font-weight: 600;
                  }
  • .quote is a selector, that defines to what particular elements of the page that particular style will apply to.
  • The styles are written in the curly braces {}
  • The font-size: 55px is a style rule.
  • font-size is the attribute and 55 px is its value.
  • Class selectors are written as .class-name.
    ID selectors are written as #id-name
    Tag selectors are written as tag-name

Adding CSS to Your Webpage

Following are the different ways to add CSS to your HTML document:
  1. Write your CSS in the <head> of the HTML document. This method is good for small projects or testing.
  2. Link a separate CSS file in your HTML document. This is the most organized method and best for larger projects.
    1. Write all of your structural HTML in one file (lets name it main.html).
    2. In a separate file (lets name it main.css), put all of your CSS styles.
    3. Add a <head> element to the top of your HTML document.
    4. Put a <link> tag inside the <head> element. Since <link> is a "void tag" you don't need to add a closing </link>.
    5. Add the following attributes to your <link rel="stylesheet" type="text/css" href="main.css">
  3. Write your CSS styles inline with your HTML document. This often results in a lot of unwanted and repeated code.

Box-Sizing

The whole web-page is made up of a number of boxes. The boxes are governed by their margin, border, padding and content. There are 2 ways to size boxes:
  1. Set sizes in terms of percentage of the screen as opposed to fixed pixels.
  2. Set the box-sizing attribute to the value border-box for every box.

Box-Positioning

  • Each <div> is a block element that takes up the entire width of the page.
  • Using the flex-box can help position the divs next to each other.
